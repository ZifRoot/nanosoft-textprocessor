# Техническое задание: Nanosoft TextProcessor

## 1. Назначение и цели разработки

Программа предназначена для обработки одного или нескольких текстовых файлов: удаления коротких слов и знаков препинания.  
Цель – предоставить пользователю удобный инструмент для очистки текстовых данных с возможностью гибкой настройки правил обработки и работы с файлами большого объёма. Ключевое требование – прозрачность процесса: пользователь должен видеть прогресс обработки (процент выполнения, текущий файл, скорость), чтобы сохранять контроль и понимать, сколько времени осталось.

## 2. Общее описание системы

Система состоит из двух основных компонентов:

- **Библиотека классов TextProcessor.Lib** – реализует всю логику обработки текста, поддерживает паттерн «Стратегия» для расширения функциональности, обеспечивает потоковое чтение/запись файлов и потоко-безопасность.
- **Графическое приложение TextProcessor.UI** (WPF) – предоставляет пользовательский интерфейс для настройки параметров, ввода списка файлов, запуска обработки и отображения прогресса.

Библиотека покрывается модульными тестами. Приложение использует асинхронные операции для сохранения отзывчивости интерфейса и регулярного обновления информации о ходе выполнения.

## 3. Функциональные требования

### 3.1. Ввод списка файлов для обработки

- Интерфейс представляет собой **таблицу (список)**, где каждая строка соответствует одному заданию на обработку и содержит:
  - Поле ввода пути к **входному файлу** (с кнопкой обзора).
  - Поле ввода пути к **выходному файлу** (с кнопкой обзора). Если поле оставлено пустым, имя формируется автоматически с использованием шаблона или суффикса, заданного в конфигурации (см. п. 5.3). По умолчанию добавляется суффикс `_processed` перед расширением.
- По умолчанию таблица содержит **одну строку**.
- Управление списком:
  - Кнопка **«+»** – добавляет новую пустую строку в конец таблицы.
  - Кнопка **«-»** – удаляет выделенную строку (или последнюю, если ничего не выделено). Если осталась только одна строка, удаление блокируется или очищает поля (на усмотрение реализации).
- Пользователь может редактировать пути как вручную, так и через диалоги выбора файлов.

### 3.2. Параметры обработки (едины для всех файлов в списке)

Все настройки располагаются на главном окне, группируются логически (например, в рамке «Параметры обработки»). Набор параметров:

- **Минимальная длина слова** (целое положительное число). Слова, длина которых **меньше** указанного числа, удаляются.
- **Удалять знаки препинания** (флажок). Если включён, становятся доступными дополнительные настройки:
  - **Режим обработки пунктуации**: выбор из трёх вариантов:
    - `Удалить` – просто удалить символ (может вызвать склейку слов, например "Привет,мир" → "Приветмир").
    - `Заменить на пробел` – всегда заменять знак препинания на пробел.
    - `Умная замена` – заменять на пробел только если знак препинания находится между двумя символами слова (буква/цифра), в остальных случаях удалять.
  - **Набор символов пунктуации** (редактируемое текстовое поле, можно добавить/удалить символы). По умолчанию: `.,!?;:—…()[]{}"'` и тире.
- **Кодировка входного файла**: выпадающий список (UTF-8, UTF-16 LE, UTF-16 BE, Windows-1251, ASCII) + пункт «Автоопределение».
- **Дополнительные лексические правила** (можно свернуть в группу «Дополнительно»):
  - `Считать цифры частью слова` (флажок, по умолчанию включён).
  - `Считать дефис частью слова` (флажок, по умолчанию включён).
  - `Считать апостроф частью слова` (флажок, по умолчанию включён).
  - `Дополнительные символы слова` (текстовое поле, можно ввести символы, которые будут считаться частью слова, например, `_@`).
- **Нормализация пробелов**: выбор из трёх вариантов:
  - `Не изменять` – оставить как есть.
  - `Схлопывать` – заменить последовательности пробельных символов на один пробел.
  - `Схлопывать и обрезать` – дополнительно удалить пробелы в начале и конце строки.
- **Сохранять переносы строк** (флажок, по умолчанию включён).
- **Шаблон имени выходного файла** (для автоматического формирования, если поле выходного файла пустое). Текстовое поле с поддержкой переменных: `{original}` (имя без расширения), `{ext}` (расширение с точкой), `{suffix}` (значение из поля «Суффикс»). По умолчанию: `{original}{suffix}{ext}`.
- **Суффикс** (текстовое поле, по умолчанию `_processed`).

Все изменения параметров применяются ко всем заданиям в списке.

### 3.3. Запуск обработки и отображение прогресса

- Кнопка **«Запустить обработку»** запускает обработку всех файлов из списка последовательно или параллельно (в зависимости от настройки параллелизма).
- Кнопка **«Отмена»** становится активной во время обработки, позволяет прервать выполнение (с подтверждением).
- **Индикация прогресса**:
  - Общий прогресс по всем заданиям: прогресс-бар, отображающий долю обработанных файлов (или байт, если можно оценить суммарный объём).
  - Для текущего обрабатываемого файла: отдельная строка с именем файла, прогресс-бар по этому файлу (процент, скорость, оставшееся время).
  - Многострочное текстовое поле с логом: какой файл начат, какие ошибки возникли, какой файл завершён.
- По окончании обработки выводится сводка: сколько файлов обработано успешно, сколько с ошибками.

### 3.4. Обработка случая совпадения входного и выходного файла

- Если для какой-либо строки входной и выходной пути совпадают (обработка «на месте»), перед запуском выводится предупреждение с предложением использовать безопасный режим (создание временного файла и атомарная замена). Пользователь может подтвердить использование безопасного режима для всех таких заданий или отменить запуск.

## 4. Нефункциональные требования

### 4.1. Производительность и работа с большими файлами

- Файлы любого размера (до десятков гигабайт) должны обрабатываться без загрузки целиком в оперативную память.
- Реализуется потоковая обработка: чтение буферами фиксированного размера, накопление данных до границ слов, обработка, запись буферами.
- Для максимальной эффективности используется асинхронное чтение/запись (`StreamReader`/`StreamWriter` с `ReadAsync`/`WriteAsync`) в сочетании с буферизацией на уровне .NET.
- Интерфейс должен регулярно обновлять информацию о прогрессе (не реже одного раза в секунду), чтобы пользователь видел динамику обработки. Это достигается использованием `IProgress<T>` и периодическим вызовом из фоновой операции.

### 4.2. Технологии и платформа

- .NET 8.0 (LTS) как основная платформа.
- Библиотека классов: стандартная библиотека .NET (net8.0).
- Пользовательский интерфейс: WPF (Windows Presentation Foundation) с использованием паттерна MVVM.
- Тестирование: xUnit или NUnit + Moq.

### 4.3. Надёжность и обработка ошибок

- Программа должна корректно обрабатывать исключительные ситуации:
  - Отсутствие прав доступа к файлу.
  - Недостаточно места на диске.
  - Некорректная кодировка.
  - Прерывание операции пользователем (кнопка «Отмена»).
- Все ошибки логируются в интерфейсе пользователя (с возможностью сохранить лог) и не приводят к аварийному завершению.

## 5. Архитектура и дизайн

### 5.1. Паттерн «Стратегия» для кастомизации обработки

Библиотека предоставляет интерфейс `ITextProcessingStrategy`, который инкапсулирует алгоритм модификации текста:

```csharp
public interface ITextProcessingStrategy
{
    string Process(string input);
}
```

Базовая реализация `LengthBasedRemovalStrategy` использует конфигурационный объект `TextProcessingOptions`, который содержит все настраиваемые лексические правила и параметры именования выходных файлов (см. раздел 5.3).

### 5.2. Определение слова и обработка границ

Правила выделения слов (параметризуются через конфигурацию):
- Слово – последовательность символов, относящихся к категориям Unicode `Letter`, `Digit` (цифры считаются частью слова) и символ дефис `-` (U+002D) внутри слова (например, «кое-как» считается одним словом). Апостроф внутри слова (например, «O’Connor») также сохраняется как часть слова.
- Возможность настройки: пользователь может включить/отключить учёт цифр как части слова, добавить дополнительные символы, считающиеся частью слова (например, подчёркивание), или исключить дефис из состава слова (тогда дефис будет считаться разделителем, как пунктуация).
- Символы пунктуации (из заданного набора) всегда являются разделителями слов и подлежат удалению.
- Особая ситуация – когда между словами стоит знак препинания без пробела (например, «Привет,мир»). Простое удаление знака приведёт к склейке слов в «Приветмир». Чтобы избежать этого, предусмотрены различные стратегии обработки пунктуации, задаваемые в конфигурации (см. п. 5.3).

### 5.3. Конфигурация обработки (лексические правила и параметры именования)

Все вопросы лексики, форматирования и именования выходных файлов решаются через единый объект конфигурации `TextProcessingOptions`, который может быть сериализован и сохранён/загружен. Основные параметры:

| Параметр | Тип | Описание | Значение по умолчанию |
|----------|-----|----------|------------------------|
| `MinWordLength` | `int` | Минимальная длина сохраняемого слова (строго меньше – удаляются) | 3 |
| `RemovePunctuation` | `bool` | Удалять ли знаки препинания | `true` |
| `PunctuationSet` | `HashSet<char>` | Набор символов, считающихся знаками препинания. Может быть расширен пользователем. | `.,!?;:—…()[]{}"'` и другие тире (например, '–', '—') |
| `PunctuationHandling` | `enum` | Определяет, что делать со знаками препинания при их удалении:<br>• `Remove` – просто удалить (может вызвать склейку слов)<br>• `ReplaceWithSpace` – всегда заменять на пробел (гарантирует отсутствие склейки, но может добавлять лишние пробелы)<br>• `SmartReplace` – заменять на пробел только если знак препинания находится непосредственно между двумя символами слова (буква/цифра); в остальных случаях удалять | `SmartReplace` |
| `KeepDigitsAsPartOfWord` | `bool` | Считать цифры частью слова (иначе цифры – разделители) | `true` |
| `KeepHyphenAsPartOfWord` | `bool` | Считать дефис внутри слова частью слова (иначе – разделитель) | `true` |
| `KeepApostropheAsPartOfWord` | `bool` | Считать апостроф внутри слова частью слова | `true` |
| `AdditionalWordChars` | `string` | Дополнительные символы, которые следует считать частью слова (например, '_', '@') | `""` |
| `WhitespaceNormalization` | `enum` | Режим нормализации пробелов:<br>• `None` – оставить как есть (может быть много пробелов)<br>• `Collapse` – заменить последовательности пробельных символов на один пробел<br>• `Trim` – дополнительно удалить пробелы в начале и конце строки | `Collapse` |
| `PreserveLineBreaks` | `bool` | Сохранять ли символы перевода строки (если `false`, строки могут объединяться) | `true` |
| `OutputFileNameTemplate` | `string` | Шаблон для формирования имени выходного файла, когда оно не задано явно. Поддерживаемые переменные:<br>• `{original}` – имя исходного файла без расширения<br>• `{ext}` – расширение исходного файла (с точкой)<br>• `{suffix}` – значение из `OutputFileSuffix`<br>Пример: `{original}{suffix}{ext}` | `{original}{suffix}{ext}` |
| `OutputFileSuffix` | `string` | Суффикс, добавляемый к имени исходного файла (используется в шаблоне по умолчанию) | `_processed` |

**Расширяемость:** Стратегия обработки получает этот объект при создании и использует его для принятия решений. В будущем можно добавлять новые параметры без изменения интерфейса стратегии.

### 5.4. Потоковая обработка файлов

Для обработки файла используется класс `TextFileProcessor`, который принимает:
- путь входного файла,
- путь выходного файла,
- стратегию обработки,
- кодировку,
- cancellation token,
- прогресс.

Он открывает `StreamReader` и `StreamWriter` с буферами и читает файл построчно (или блоками с учётом возможного разрыва слов на границе буфера).  
Для отслеживания прогресса используется общее количество байт/строк (если размер файла известен) или приблизительная оценка.

### 5.5. Многопоточность в пакетной обработке

Пакетная обработка управляется сервисом `BatchProcessor`, который:
- Получает список файлов.
- Создаёт ограниченное количество параллельных задач.
- Прогресс по каждому файлу собирается и агрегируется.
- Степень параллелизма настраивается пользователем в интерфейсе.

## 6. Требования к пользовательскому интерфейсу (WPF)

### 6.1. Главное окно (единственное)

Окно условно делится на три области: список файлов, параметры обработки, кнопки управления и прогресс.

#### 6.1.1. Список файлов (таблица)

- Заголовки столбцов: «Входной файл», «Выходной файл».
- Каждая строка содержит:
  - Текстовое поле для пути входного файла.
  - Кнопку «…» рядом с ним для выбора файла через диалог.
  - Текстовое поле для пути выходного файла.
  - Кнопку «…» для выбора файла/папки (можно разрешить выбор только файла).
- Под таблицей расположены кнопки:
  - **«+»** (Добавить строку) — добавляет новую пустую строку.
  - **«-»** (Удалить строку) — удаляет текущую выбранную строку. Если ни одна строка не выбрана, удаляется последняя. Если осталась одна строка, кнопка блокируется или очищает поля.

#### 6.1.2. Параметры обработки

Все элементы управления сгруппированы визуально (например, в GroupBox «Параметры обработки»). Рекомендуется использовать вкладки или разворачиваемые группы для компактности, но все параметры должны быть видны без дополнительных модальных окон.

- **Минимальная длина слова**: `NumericUpDown` (целые, от 1 до 100).
- **Удалять знаки препинания**: `CheckBox`. При снятии чекбокса связанные с пунктуацией элементы могут отключаться.
  - **Режим обработки пунктуации**: `ComboBox` с вариантами: «Удалить», «Заменить на пробел», «Умная замена».
  - **Набор символов пунктуации**: `TextBox` (многострочный или однострочный) с возможностью редактирования. Рядом кнопка «Восстановить умолчания».
- **Кодировка входного файла**: `ComboBox` со списком: UTF-8, UTF-16 LE, UTF-16 BE, Windows-1251, ASCII, Автоопределение.
- **Дополнительно** (можно вынести в группу «Лексические правила»):
  - `CheckBox` «Цифры — часть слова».
  - `CheckBox` «Дефис — часть слова».
  - `CheckBox` «Апостроф — часть слова».
  - `TextBox` «Доп. символы слова» (подпись: символы, которые считать частью слова, например, _@).
- **Нормализация пробелов**: `ComboBox` с вариантами: «Не изменять», «Схлопывать», «Схлопывать и обрезать».
- **Сохранять переносы строк**: `CheckBox`.
- **Формирование имени выходного файла** (если поле не заполнено):
  - `TextBox` «Шаблон» с подсказкой о переменных `{original}`, `{ext}`, `{suffix}`.
  - `TextBox` «Суффикс» (по умолчанию `_processed`).

#### 6.1.3. Управление и прогресс

- Кнопка **«Запустить обработку»** — запускает обработку всех заданий из таблицы с текущими параметрами.
- Кнопка **«Отмена»** — активна во время обработки, прерывает выполнение.
- **Область прогресса**:
  - Прогресс-бар общего прогресса (по всем файлам).
  - Текстовое поле (или список) с логом: сообщения о начале обработки каждого файла, об ошибках, об успешном завершении.
  - Для текущего файла: дополнительный прогресс-бар (или детализация в логе), отображение имени файла, процентов, скорости, оставшегося времени.
- Статусная строка (например, «Готов», «Обработка...», «Завершено»).

### 6.2. Поведение при длительных операциях

- Все длительные операции запускаются асинхронно с использованием `async/await` и `CancellationToken`.
- Интерфейс остаётся отзывчивым, кнопка «Отмена» прерывает обработку (с подтверждением).
- При отмене временные файлы удаляются, если использовался безопасный режим.

## 7. Требования к тестированию

### 7.1. Модульные тесты для библиотеки TextProcessor.Lib

- **Тестирование стратегии с различными конфигурациями** (проверка всех параметров из `TextProcessingOptions`, включая `PunctuationHandling` и `OutputFileNameTemplate`).
- **Тестирование потокового процессора** (чтение/запись, обработка больших данных, отмена).
- **Тестирование вспомогательных классов**.

### 7.2. Интеграционное тестирование

- Проверка пакетной обработки нескольких файлов с разными параметрами.
- Проверка параллельной обработки с ограничением потоков.
- Проверка безопасной обработки при совпадении входного и выходного файла.
- Проверка корректного формирования имён выходных файлов по шаблону.

## 8. Примеры работы

### Пример 1 (один файл, удаление слов короче 4, удаление знаков препинания, конфигурация по умолчанию)

**Таблица заданий:**
| Входной файл | Выходной файл |
|--------------|---------------|
| input.txt    | (пусто)       |

**Параметры:** MinLength=4, RemovePunctuation=true, кодировка UTF-8, остальное по умолчанию (PunctuationHandling=SmartReplace).

**Результат:** создаётся файл `input_processed.txt` с обработанным текстом.

### Пример 2 (два файла, разные выходные имена)

**Таблица заданий:**
| Входной файл | Выходной файл      |
|--------------|--------------------|
| data1.txt    | result1.txt        |
| data2.txt    | (пусто)            |

**Параметры:** MinLength=3, RemovePunctuation=true, шаблон `{original}_clean{ext}`, суффикс `_processed` (не используется, так как шаблон явный).

**Результат:** 
- `data1.txt` → `result1.txt`
- `data2.txt` → `data2_clean.txt`

### Пример 3 (обработка с умной заменой пунктуации)

**Входной файл `test.txt`:** `Привет,мир`
**Параметры:** RemovePunctuation=true, PunctuationHandling=SmartReplace.
**Выходной файл:** `Привет мир` (запятая заменена на пробел).

Если бы был выбран режим `Remove`, результат был бы `Приветмир` (склейка).

### Пример 4 (пакетная обработка с настраиваемым суффиксом)

**Параметры конфигурации:** OutputFileNameTemplate = `{original}_clean{ext}`, OutputFileSuffix не используется (шаблон явный).

Входной файл `data.txt` → выходной `data_clean.txt`.

## 9. Критерии приемки

1. Программа корректно обрабатывает текстовые файлы согласно заданным правилам (проверяется набором тестовых примеров).
2. Пользователь видит понятный прогресс обработки (процент, скорость, оставшееся время) в реальном времени.
3. Пакетная обработка нескольких файлов выполняется параллельно и существенно быстрее последовательной.
4. Все модульные тесты проходят успешно.
5. Приложение корректно обрабатывает ситуацию совпадения входного и выходного файла, предлагая безопасные варианты.
6. Настройки лексических правил и именования выходных файлов доступны и применяются.
7. Приложение не вызывает исключений при корректных действиях пользователя и корректно обрабатывает ошибочные ситуации.
